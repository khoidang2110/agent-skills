# skill-react
This skill assumes a Tailwind-based or utility-first styling system.

UI consistency playbook for **React** projects.
## Purpose
- Keep styling consistent across screens and components
- Encourage reuse of existing UI primitives
- Reduce one-off styles and visual drift
- Make UI changes predictable for humans and AI agents

## Scope
This skill governs:
- Styling conventions (tokens/scale/system)
- Component reuse vs creation rules
- Interaction states (hover/focus/disabled)
- Dark mode parity (if the project supports it)

Not included:
- Backend/API architecture rules
